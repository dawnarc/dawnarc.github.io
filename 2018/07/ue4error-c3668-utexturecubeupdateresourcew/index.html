Caused by:
Maybe there are some third party libraries that using Windows API were linked in your game, and these libraries are conflicted with Windows SDK which your codes (e.g. using CreateMutex() of VC++, not std::mutex of STL) dependented on in local environment.

Solution:
On Unreal Forums, there is a saying: include following headers in MyProject.h in your game.

#include "UMG.h"
#include "UMGStyle.h"
#include "Slate/SObjectWidget.h"
#include "IUMGModule.h"
#include "Blueprint/UserWidget.h"

Reference:
https://forums.unrealengine.com/development-discussion/c-gameplay-programming/113028-getting-build-error-in-my-engine-plugin-while-building-in-ue4-editor-while-packaging-the-project

But sometimes it doesn’t work even these headers included, the silver bullet is to replace Windows API with cross platform libraries (e.g. STL) in your game, e.g. instead of CreateMutex() with std::mutex, or make sure third party libraries didn’t link Windows libraries statically, but rather linking dynamically.
