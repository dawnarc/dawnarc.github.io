[OpenGL]Rendering Pipeline(渲染管线)流程概述

Sunday, 20:58, Jun 24, 2018 in OpenGL
Pipeline (The blue boxes are programmable shader stages.) The OpenGL rendering pipeline works in the following order: Prepare vertex array data, and then render it Vertex Processing: Each vertex is acted upon by a Vertex Shader. Each vertex in the stream is processed in turn into an output vertex. Optional primitive tessellation stages. Optional Geometry Shader primitive processing. The output is a sequence of primitives. Vertex Post-Processing, the outputs of the last stage are adjusted or shipped to different locations.
